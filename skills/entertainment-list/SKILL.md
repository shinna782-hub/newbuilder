# 娱乐清单 - 娱乐作品记录

记录你想看、正在看、或者已经看完的娱乐作品。

> 状态：早期版本。分类选项带有个人偏好，首次使用前建议按自己的表格调整。

## 触发规则
- 触发指令：`娱乐`。它用于开启一次娱乐清单整理任务，不要求每条后续消息重复使用。
- 用户使用 `娱乐` 开启任务后，只要后续上下文仍在整理娱乐作品，就持续调用本技能。
- 在名为「娱乐清单整理」的专属群中，如果消息内容是作品名称、观看状态、分类、备注或对上一条记录的补充，默认按本技能处理，无需触发指令。
- 用户明确表示“结束整理”、切换任务或上下文已经明显变为其他话题时，退出本技能；之后需要再次使用 `娱乐` 或明确提出整理娱乐清单，才能重新进入。
- 在其他群聊或私聊中，若当前上下文没有明确处于娱乐清单整理任务，不应仅凭普通聊天内容调用本技能。
- 飞书群聊、飞书私聊和微信均遵循以上上下文延续规则。

## 目标多维表格
- App Token: 环境变量 `ENTERTAINMENT_LIST_BITABLE_APP_TOKEN`
- Table ID: 环境变量 `ENTERTAINMENT_LIST_BITABLE_TABLE_ID`
- 名称：娱乐清单

## 分类自动推断规则

### 一级分类
根据内容类型推断：
- **动画/综艺**（非动画电影） → 线上节目
- **真人电影**（科幻/国产/超英/国外其他/长剧） → 线上节目
- **动画电影** → 向往专家（此分类尚未开放，暂填"线上节目"）

>⚠️ 只有动画电影才属于"向往专家"。目前用户尚未发送动画电影数据，暂统一填"线上节目"。

### 二级分类
根据用户指定的三级分类推断：
- 三级分类为动画类（中国动画/日本动画/他国动画/耽美动画）→ 动画
- 三级分类为综艺类 → 综艺
- 三级分类为电影类（科幻/国产/超英/国外其他）→ 电影
- 三级分类为长剧类 → 长剧
- 三级分类为漫画类 → 漫画

### 三级分类
根据用户输入的作品信息判断，例如：
- "想看《鬼灭之刃》" → 日本动画
- "《庆余年》" → 国产剧
- "推荐《星际穿越》" → 科幻电影

### 追更状态
从语境判断：
- 用户说"想看/推荐" → 高优想看
- 用户说"在看/追" → 正在看
- 用户说"看完了/已看完" → 已看完
- 用户说"弃了/弃番/看一半" → 看一半弃了
- 默认状态 → 未启动

## 工作流程

### Step 1: 解析用户输入
提取：
- 作品名称
- 三级分类（根据作品判断）
- 添加原因/备注
- 追更状态（从语境判断）
- 系列作品备注（如有续作/多季）

### Step 2: 补全分类
根据三级分类自动推断二级分类，根据二级分类自动推断一级分类。

### Step 3: 获取 Feishu Token
```bash
: "${FEISHU_APP_ID:?FEISHU_APP_ID is not set}"
: "${FEISHU_APP_SECRET:?FEISHU_APP_SECRET is not set}"
curl -s -X POST 'https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal' \
  -H 'Content-Type: application/json' \
  -d "{\"app_id\":\"${FEISHU_APP_ID}\",\"app_secret\":\"${FEISHU_APP_SECRET}\"}"
```

### Step 4: 创建记录
```bash
: "${ENTERTAINMENT_LIST_BITABLE_APP_TOKEN:?ENTERTAINMENT_LIST_BITABLE_APP_TOKEN is not set}"
: "${ENTERTAINMENT_LIST_BITABLE_TABLE_ID:?ENTERTAINMENT_LIST_BITABLE_TABLE_ID is not set}"
curl -s -X POST "https://open.feishu.cn/open-apis/bitable/v1/apps/${ENTERTAINMENT_LIST_BITABLE_APP_TOKEN}/tables/${ENTERTAINMENT_LIST_BITABLE_TABLE_ID}/records" \
  -H 'Authorization: Bearer <token>' \
  -H 'Content-Type: application/json' \
  -d '{
    "fields": {
      "作品名称": "<作品名称>",
      "一级分类": "<一级分类>",
      "二级分类": "<二级分类>",
      "三级分类": "<三级分类>",
      "添加原因/备注": "<原因或备注>",
      "系列作品备注": "<系列备注>",
      "追更状态": "<状态>",
      "添加时间": "<当前时间戳>"
    }
  }'
```

### Step 5: 返回结果
写入完成后返回给用户确认：
- 作品名称
- 三级分类
- 追更状态

## 使用示例

> 娱乐 《鬼灭之刃》很好看，想看

→写入：作品名称《鬼灭之刃》，一级分类「线上节目」，二级分类「动画」，三级分类「日本动画」，添加原因/备注「很好看」，追更状态「高优想看」

> 娱乐 推荐《星际穿越》，太震撼了

→ 写入：作品名称《星际穿越》，一级分类「线上节目」，二级分类「电影」，三级分类「科幻电影」，添加原因/备注「太震撼了」，追更状态「高优想看」

> 娱乐 《庆余年》第三季什么时候出

→ 写入：作品名称《庆余年》，一级分类「线上节目」，二级分类「长剧」，三级分类「国产剧」，系列作品备注「第三季」，追更状态「未启动」

## 注意事项
- 一部作品对应一条记录；一条消息包含多部作品时，必须拆分并为每部作品分别创建记录
- 如果作品已存在（同名），提示用户是否更新现有记录
- 三级分类无法判断时，优先根据作品名查证；仍无法判断则向用户确认，不得把“未启动”等追更状态写入三级分类
- 分类选项以飞书多维表格实际配置为准
